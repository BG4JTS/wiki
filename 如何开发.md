
## 项目名称
**原样WIKI** —— 一个基于文字稿的播客讨论平台。

## 核心约束
- **不存储音频文件**：仅处理文稿、时间轴、评论和用户贡献。
- **零成本运行**：完全依赖免费层，月成本趋近于 0。
- **数据规模**：500 期节目，每期约 8000-10000 字中文，共约 500 万字。

## 技术栈 (Tech Stack)
- **前端/后端托管**: Vercel (Hobby Plan)。
- **框架**: Next.js (使用 App Router)。
- **数据库与认证**: Supabase (Free Plan)。
- **样式**: Tailwind CSS。
- **类型安全**: TypeScript。

## 架构模式
**BFF (Backend-for-Frontend) 极简模式**：
- Next.js 主要负责**静态生成 (SSG)** 和**服务端渲染 (SSR)** 页面，以便 SEO。
- **禁止**编写传统的 API Routes 来处理增删改查。
- 所有数据交互（增、删、改、查）**直接**通过 Supabase JavaScript 客户端在前端完成。
- 唯一例外：Supabase 行级安全策略处理权限；如需环境变量保护逻辑，使用 Server Component 或 Server Action。

## 数据模型 (Supabase Schema)
请严格按照以下 SQL 结构创建表：

1.  **episodes** (节目主表)
    -   `id` (int8, 主键)
    -   `episode_number` (int4, 唯一, 索引)
    -   `title` (text)
    -   `publish_date` (date)
    -   `duration` (int4, 备注：时长秒数)
    -   `description` (text)
    -   `transcript` (text, 备注：长文本, 存储完整文稿)
    -   `created_at` (timestamptz)

2.  **timelines** (时间轴)
    -   `id` (int8, 主键)
    -   `episode_id` (int8, 外键关联 episodes.id, 级联删除)
    -   `timestamp_sec` (int4, 备注：时间点秒数)
    -   `title` (varchar(100), 备注：章节标题)
    -   `description` (text, 备注：该时间点说明)

3.  **comments** (评论)
    -   `id` (int8, 主键)
    -   `episode_id` (int8, 外键)
    -   `user_id` (uuid, 外键关联 auth.users)
    -   `content` (text)
    -   `parent_id` (int8, 自关联外键, 备注：支持嵌套回复)
    -   `likes` (int4, 默认 0)
    -   `created_at` (timestamptz)

4.  **user_profiles** (用户扩展信息)
    -   `id` (uuid, 主键, 关联 auth.users)
    -   `username` (text, 唯一)
    -   `avatar_url` (text, 备注：可使用默认头像或 DiceBear API)
    -   `bio` (text)

5.  **user_contributions** (用户补充信息)
    -   `id` (int8, 主键)
    -   `episode_id` (int8, 外键)
    -   `user_id` (uuid, 外键)
    -   `content_type` (text, 备注：限制为 'link', 'note', 'image_url')
    -   `content` (text, 备注：存储 URL 或纯文本)
    -   `description` (varchar(500))
    -   `votes` (int4, 默认 0)
    -   `created_at` (timestamptz)

## 安全与访问规则 (Supabase RLS)
必须创建行级安全策略：

1.  **episodes & timelines**：
    -   `SELECT`：允许 `true` (任何人可读)。
    -   `INSERT/UPDATE/DELETE`：仅允许 `auth.email() = '你的管理员邮箱'`。

2.  **comments**：
    -   `SELECT`：允许 `true`。
    -   `INSERT`：允许 `auth.role() = 'authenticated'`，且 `user_id` 必须等于 `auth.uid()`。
    -   `UPDATE/DELETE`：仅允许 `auth.uid() = user_id`。

3.  **user_profiles**：
    -   `SELECT`：允许 `true`。
    -   `INSERT/UPDATE`：仅允许 `auth.uid() = id` (用户只能修改自己的)。
    -   在应用层配合触发器：用户注册时自动创建默认 Profile。

4.  **user_contributions**：
    -   `SELECT`：允许 `true`。
    -   `INSERT`：允许 `auth.role() = 'authenticated'`。

## 页面路由与功能 (App Router)
-   `/` (首页)：静态生成。展示分页的节目列表 (`episodes`)，按 `episode_number` 倒序。
-   `/episodes/[id]` (详情页)：静态生成 + 增量静态再生成。展示标题、描述、时长、**全文文稿 (支持高亮关键词)**、**时间轴 (可点击跳转，虽无音频播放器，但需显示时间点标记)**、评论区 (树形结构/嵌套回复)、用户贡献区 (链接/笔记/图片)。
-   `/login`：客户端组件。包含邮箱/密码登录和注册表单。
-   `/profile`：服务端组件(需鉴权)。展示用户信息、修改用户名/简介、查看自己的评论历史。
-   `/admin` (可选)：严格鉴权 (需验证邮箱)。仅允许管理员使用的富文本编辑器，用于录入/编辑 `episodes` 和 `timelines`。

## 关键交互逻辑
-   **全文搜索**：在导航栏提供搜索框。前端调用 Supabase 的 `textSearch` 功能，在 `episodes.title` 和 `transcript` 字段中进行搜索。
-   **无需后端**：所有 `INSERT`, `SELECT`, `DELETE` 操作均在前端使用 `supabase-js` 客户端直接执行。
-   **认证状态管理**：使用 Supabase 提供的 `useUser` 或 `createClientComponentClient` 配合 `onAuthStateChange` 监听登录状态。

## 环境变量配置 (Vercel / .env.local)
-   `NEXT_PUBLIC_SUPABASE_URL`：你的 Supabase 项目 URL。
-   `NEXT_PUBLIC_SUPABASE_ANON_KEY`：你的 Supabase 匿名密钥。
-   `ADMIN_EMAIL` (可选，用于 RLS 策略中的硬编码管理员邮箱)。

## 项目初始化步骤 (引导 Codex)
1.  创建 Next.js 项目：`npx create-next-app@latest --typescript --tailwind --app`
2.  安装依赖：`npm install @supabase/supabase-js @supabase/ssr`
3.  在 Supabase 控制台运行上述 SQL 语句建立表结构和 RLS 策略。
4.  在 `lib/supabase` 目录下创建客户端实例 (区分 Server Component 和 Client Component)。
5.  在 `middleware.ts` 中配置 Supabase 的会话刷新逻辑。
6.  按照上述路由结构生成页面文件。

## 样式与用户体验要求
-   响应式设计 (Tailwind)。
-   文稿区域需支持长文本滚动，并保留段落格式 (white-space: pre-wrap)。
-   评论嵌套需有视觉缩进。
